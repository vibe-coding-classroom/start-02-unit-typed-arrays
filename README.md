針對 **`start-02-unit-typed-arrays` (數據解析與類型化陣列 Typed Arrays)** 單元，這是學員從「標籤化數據」邁向「位元級操作」的技術分水嶺。它要求學員具備 **底層數據處理 (Binary Data Manipulation)** 的意識，學習如何跳出 JSON 的舒適圈，利用 `ArrayBuffer` 與 `TypedArray` 進行記憶體級別的操作，並掌握「位元組順序 (Endianness)」這項跨平台通訊的必備知識。

### 1. 範本倉庫 (Template Repo) 配置建議
二進位資料作業的核心在於「協定封裝與數據還原」，範本應包含通訊解析器框架與邊界測試，建議包含：
*   **📂 `index.html`**：**數據視覺化偵錯介面**。預設包含一個 Hex 預覽區塊，讓學員在調整滑桿時，能即時看到內存中的 Byte 變化。
*   **📂 `parser.js`**：**封裝器與解碼器實作區**。學員在此實作 `encodeMotorSpeed()` 與 `decodeSensorData()`。預留 `// TODO` 讓學員練習邊界補捉。
*   **📂 `README.md`**：**自訂通訊協定規範 (Protocol Spec)**。提供模板要求學員寫下：我的馬達指令中，Index 0 代表哪一輪？Index 1 代表哪一輪？
*   **📂 `test/`**：**自動化單元測試**。提供基礎測試腳本（如使用 Jest 或簡單 Assert），自動驗證學員產出的 Buffer 長度與內容是否符合規格。

### 2. 作業任務部署細節

#### 任務 1：雙通道馬達指令封裝 (Binary Encoder Lab)
*   **目標**：將抽象的速度指令封裝為硬體可理解的 2-byte Payload。掌握 Uint8 範圍限制與索引順序。
*   **Classroom 部署建議**：
    *   **封包邏輯實作**：建立一個函式，將左右馬達速度 (0-255) 依序寫入一個 `Uint8Array(2)`。
    *   **數值範圍保護**：要求實作「鉗位 (Clamping)」邏輯，當輸入負數或大於 255 時，必須安全地處理（如強制降為 0 或升為 255）。
    *   **驗證要點**：**關鍵字節審核**。學員產出的 Buffer 是否精確為兩個位元組？左右輪的順序是否與其 README 紀錄的一致？

#### 任務 2：還原感測器的 16-bit 數值 (Data Decoder & Endianness Lab)
*   **目標**：破解跨平台通訊的核心陷阱。學會利用 `DataView` 同時處理高低位元組並應對不同的端序 (Endianness)。
*   **Classroom 部署建議**：
    *   **解碼器實作挑戰**：給予一個 16-bit 的十六進位值（如 `0x03E8` 代表距離 1000mm），學員需實作代碼還原成十進位整數。
    *   **端序切換練習**：要求學員分別讀取大端序 (Big-Endian) 與小端序 (Little-Endian) 下的數值，並解釋其結果差異（如 1000 vs 59395）。
    *   **驗證要點**：是否正確使用了 `view.getUint16(0, true/false)`？學員是否能在報告中清楚說明什麼時候該開啟 Little-Endian 模式？

### 3. 位元組精準度與封包效率點評標準 (Binary Precision & Efficiency Standards)
此單元的價值在於 **「對數值位元的極致簡潔與對底層兼容性的判斷力」**：
*   [ ] **封包穩定性**：產出的 ArrayBuffer 格式是否穩定，沒有出現不必要的動態長度變化？
*   [ ] **邊界安全性**：面對異常的輸入數據，封裝器是否展現出預期的健壯性（Clamping 或是報錯提示）？
*   [ ] **端序邏輯清晰度**：學員對於「高位在前」還是「低位在前」是否有明確的設計思維而非猜測？
